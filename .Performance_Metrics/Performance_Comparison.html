Key Findings & Recommendations

Critical Finding: Archive Extraction Method

UNC path incompatibility with .NET ZipFile forces .NET Framework to fall back to PowerShell Expand-Archive cmdlet, which is 10x slower. This single factor accounts for 1m 12s of the 5m 10s improvement.

Production Deployment Best Practices

‚≠ê CRITICAL: Use Local Installation Source
  • Copy ZIP file to local temp directory before installation
  • Saves 5 minutes 10 seconds (77% faster installation)
  • Example workflow:
Copy-Item "\\NAS01\DataShare\Get-CiscoTechSupport.zip" -Destination "C:\Temp\"
.\Install-GetCiscoTechSupport.ps1 -ArchivePath "C:\Temp\Get-CiscoTechSupport.zip"
‚≠ê RECOMMENDED: Silent Installation for Automation
  • Eliminates user-interactive delays (~1 minute saved)
  • Use parameters for all configuration
  • Suitable for large-scale deployments
.\Install-GetCiscoTechSupport.ps1 `
    -ArchivePath "C:\Temp\Get-CiscoTechSupport.zip" `
    -ServiceAccountCredential $svcCred `
    -EnableEmail `
    -SMTPServer "mail.local" `
    -SMTPCredential $smtpCred
Pre-Stage Installation Files
  • For large-scale deployments, pre-copy ZIP to target machines
  • Use GPO or deployment tool to stage files locally
  • Run installation from local path
  • Achieves 5m 10s reduction per machine
Optimize Network Shares (if UNC path required)
  • Enable SMB multichannel
  • Ensure SMB3 protocol is available
  • Verify network path has sufficient bandwidth
  • Note: Still expect 5+ minute penalty vs local installation
